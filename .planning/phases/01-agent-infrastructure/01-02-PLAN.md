Add these endpoints under a new comment section `# ===========================\n# Agent Management\n# ===========================`:

**1. POST /api/agents/create** (authenticated, requires store ownership):
- Accept `AgentCreateRequest` body
- Verify store exists and has no active agent (if `agent_status` is already 'active', return 409 Conflict)
- Fetch template from DB: `SELECT conversation_config, platform_settings FROM agent_templates WHERE type = $1 AND is_default = TRUE`
- If no template found, return 404
- Set `agent_status = 'pending'` in DB
- Call `elevenlabs_service.create_agent(name=f"SimplifyOps - {shop_domain}", conversation_config=config, platform_settings=platform)`
- On success: update DB with `elevenlabs_agent_id`, `agent_status = 'active'`, `agent_config`, `agent_template_id`
- On failure: set `agent_status = 'failed'`, re-raise as 502
- Return `AgentCreateResponse`

**2. GET /api/agents/{store_id}** (authenticated, requires store ownership):
- Query DB for agent columns: `SELECT elevenlabs_agent_id, agent_status, agent_config, agent_template_id, minutes_used FROM stores WHERE id = $1::uuid`
- If store not found, return 404
- Look up template type from agent_template_id if set
- Return `AgentInfo`

**3. PATCH /api/agents/{store_id}** (authenticated, requires store ownership):
- Accept `AgentUpdateRequest` body
- Verify store has an active agent (agent_status = 'active' and elevenlabs_agent_id is not null), else 404
- Build ElevenLabs update payload from non-None fields:
  - voice_id -> `{"tts": {"voice_id": voice_id}}`
  - greeting -> `{"agent": {"first_message": greeting}}`
  - language -> `{"agent": {"language": language}}`
  - system_prompt -> `{"agent": {"prompt": {"prompt": system_prompt}}}`
  - max_duration_seconds -> `{"conversation": {"max_duration_seconds": max_duration_seconds}}`
  - Merge these into a single `conversation_config` dict
- Call `elevenlabs_service.update_agent(agent_id, conversation_config=merged_config)` FIRST
- On success, update DB `agent_config` with the merged config: `UPDATE stores SET agent_config = agent_config || $1::jsonb, updated_at = NOW() WHERE id = $2::uuid`
- On failure, return 502 without modifying DB (update ElevenLabs FIRST pattern per research pitfall #4)
- Return 200 with updated AgentInfo

**4. DELETE /api/agents/{store_id}** (authenticated, requires store ownership):
- Verify store has an agent (elevenlabs_agent_id is not null), else 404
- Call `elevenlabs_service.delete_agent(agent_id)` -- log warning if fails but continue
- Clean up DB: set `elevenlabs_agent_id = NULL, agent_status = 'none', agent_config = '{}', agent_template_id = NULL`
- Return `AgentDeleteResponse`

**5. GET /api/agents/templates** (authenticated, no store ownership needed):
- Query: `SELECT id, name, type, description, conversation_config, platform_settings, is_default FROM agent_templates ORDER BY type`
- Return list of `AgentTemplateInfo`

**6. Refactor GET /api/voice/signed-url** (existing endpoint):
- Change logic: if `store_id` is provided, look up `elevenlabs_agent_id` and `agent_status` from DB
- If store has active agent, use per-store `agent_id`
- If `agent_status != 'active'`, return 503 "Agent is not active"
- If no `store_id` provided OR store has no agent, fall back to global `settings.ELEVENLABS_AGENT_ID` (backward compatibility)
- Keep the rest of the signed URL generation the same

**7. Refactor GET /api/voice/config** (existing endpoint):
- Same pattern: if `store_id` provided, resolve per-store agent_id from DB
- Fallback to global `settings.ELEVENLABS_AGENT_ID` if no store_id or no per-store agent

IMPORTANT: Use `json.loads()` / `json.dumps()` for JSONB handling -- asyncpg returns JSONB as strings. Parse template configs with `json.loads()` if they are strings, pass through if already dicts.

IMPORTANT: All agent management endpoints (1-4) require authentication via `require_store_owner` (the existing auth pattern in main.py). Template listing (5) only requires `get_authenticated_user`. Signed URL and voice config remain public (no auth -- the widget calls these).
