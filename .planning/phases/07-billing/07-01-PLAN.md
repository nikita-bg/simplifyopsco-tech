**1. Fix GET /api/stores/{store_id}/subscription endpoint** (around line 1838 in main.py):

- Use correct TIER_LIMITS: `{"trial": 30, "starter": 100, "growth": 400, "scale": 2000}` (current code has wrong values 300/1000)
- Remove the conversations COUNT query (lines ~1865-1871) — usage is already tracked in `stores.minutes_used`
- Add `trial_ends_at` to the response (ISO string or null)
- Add `is_trial_expired` boolean: True if tier=="trial" AND trial_ends_at is not None AND trial_ends_at < NOW()

```python
    "is_trial_expired": bool,
```

**2. Add invoice.paid webhook handler** in stripe_service.py `handle_webhook_event`:
- Add `elif event_type == "invoice.paid":` block
- Extract `subscription` from data, look up store by `stripe_subscription_id`
- Reset `minutes_used = 0` and update `billing_period_start = NOW()` for that store
- Log: `SecurityLogger.log(f"Billing cycle reset: sub={subscription_id}", "INFO")`
- If the store has `agent_status = 'limit_reached'`, restore it to `'active'` (re-enable after new cycle)

**3. Extract TIER_LIMITS to a shared constant** in stripe_service.py (module level):

```python
    "starter": 100,
    "growth": 400,
    "scale": 2000,
}
```
This is needed so both stripe_service.py and automation_service.py can import from the same source. Update automation_service.py to import: `from backend.stripe_service import TIER_LIMITS` and remove its local `TIER_LIMITS` class attribute.

**4. Add 110% enforcement in post-call webhook** (main.py around line 453 where minutes_used is incremented):
After the `minutes_used` UPDATE, add a check:
```python
# Check if store exceeded 110% of tier limit
row = await db.fetchrow(
    "SELECT subscription_tier, minutes_used FROM stores WHERE id = $1::uuid", store_id
)
if row:
    from backend.stripe_service import TIER_LIMITS
    limit = TIER_LIMITS.get(row["subscription_tier"] or "trial", 30)
    if (row["minutes_used"] or 0) >= limit * 1.1:
        await db.execute(
            "UPDATE stores SET agent_status = 'limit_reached' WHERE id = $1::uuid AND agent_status = 'active'",
            store_id
        )
        SecurityLogger.log(f"Usage limit reached: store={store_id} minutes={row['minutes_used']}/{limit}", "WARNING")
```
