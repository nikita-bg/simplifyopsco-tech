  <name>Task 1: Add onboarding_step column, enhance store creation, and build status endpoint</name>
  <files>backend/main.py</files>
  <action>
1. **Add DB migration in ensure_tables()**: Add `onboarding_step` TEXT column to stores table (default 'none'). Valid values: 'none', 'pending', 'creating_agent', 'syncing_kb', 'sending_email', 'complete', 'failed'. Also add `onboarding_error` TEXT column (nullable) for error messages, and `store_name` TEXT column (nullable) for the merchant's chosen store name.

2. **Enhance POST /api/stores/create** (line ~1708):
   - Accept additional fields from body: `store_name` (string, optional, max 100 chars), `store_type` (string, optional, one of 'online_store', 'service_business', 'lead_gen', defaults to 'online_store')
   - Validate store_type against allowed values, return 400 if invalid
   - INSERT store_name into stores table
   - Set onboarding_step to 'pending' on insert
   - Pass store_type to run_onboarding as a new parameter
   - Return store_name and store_type in response alongside existing fields

3. **Add GET /api/stores/{store_id}/onboarding-status** endpoint:
   - Requires authentication (use get_authenticated_user, then verify owner_id matches)
   - SELECT onboarding_step, onboarding_error, agent_status, store_name, elevenlabs_agent_id FROM stores WHERE id = $1
   - Return JSON:
     ```json
     {
       "store_id": "...",
       "step": "creating_agent",
       "completed_steps": ["pending", "creating_agent"],
       "is_complete": false,
       "is_failed": false,
       "error": null,
       "has_agent": false
     }
     ```
   - `completed_steps` is derived: all steps up to and including current step in the sequence
   - `is_complete` = step == 'complete'
   - `is_failed` = step == 'failed'
   - `has_agent` = elevenlabs_agent_id is not null AND agent_status == 'active'
   - If store not found, return 404

<task type="auto">
  <name>Task 2: Update run_onboarding to track step progress and accept store_type</name>
  <files>backend/automation_service.py, backend/tests/test_onboarding.py</files>
  <action>
1. **Update run_onboarding signature** to accept `store_type: str = "online_store"` parameter.

2. **Add step tracking** via DB updates at each stage of run_onboarding:
   - Before agent creation: `UPDATE stores SET onboarding_step = 'creating_agent' WHERE id = $1::uuid`
   - Before KB sync: `UPDATE stores SET onboarding_step = 'syncing_kb' WHERE id = $1::uuid`
   - Before welcome email: `UPDATE stores SET onboarding_step = 'sending_email' WHERE id = $1::uuid`
   - After all complete: `UPDATE stores SET onboarding_step = 'complete' WHERE id = $1::uuid`
   - On agent creation failure: `UPDATE stores SET onboarding_step = 'failed', onboarding_error = 'Agent creation failed: {str(exc)}' WHERE id = $1::uuid`
   - On any top-level exception: `UPDATE stores SET onboarding_step = 'failed', onboarding_error = '{str(exc)}' WHERE id = $1::uuid`

3. **Use store_type for template lookup**: Change the template query from hardcoded `type = 'online_store'` to `type = $2` with the store_type parameter. This ensures the correct template (online_store, service_business, lead_gen) is selected based on the merchant's form choice.
