       - Class `EmailService` with `__init__` that stores `api_key` and `from_email` from settings.
       - Method `async send_welcome_email(self, to_email: str, store_domain: str, store_id: str)`:
         - Uses httpx POST to `https://api.resend.com/emails` with Authorization header `Bearer {api_key}`

       - Import `apscheduler.schedulers.asyncio.AsyncIOScheduler` (APScheduler 4.x async-native)
       - Class `AutomationService` with:
         - `self.scheduler: AsyncIOScheduler` initialized in `__init__`
         - `async start(self)`: calls `self.scheduler.start()` — scheduler runs jobs in the background
         - `async stop(self)`: calls `self.scheduler.shutdown()`
         - `async run_onboarding(self, store_id: str, owner_email: str | None = None)`:
           a. Log "Starting onboarding for store {store_id}"
           b. Fetch store row (shop_domain, agent_status) from DB
           c. Fetch default template for 'online_store'
           d. Create ElevenLabs agent via elevenlabs_service.create_agent (catch errors, set agent_status='failed', log, return early — DO NOT send welcome email on failure)
           e. Update DB: set elevenlabs_agent_id, agent_status='active', agent_config, agent_template_id
           f. Try KB sync via kb_service.trigger_kb_rebuild (catch errors, log warning, continue — KB sync failure should not block onboarding)
           g. Send welcome email via email_service.send_welcome_email if owner_email is provided
           h. Log "Onboarding complete for store {store_id}"
           i. Wrap entire method in try/except at top level — log error, never let exception propagate to crash BackgroundTasks
       - Module-level singleton: `automation_service = AutomationService()`
