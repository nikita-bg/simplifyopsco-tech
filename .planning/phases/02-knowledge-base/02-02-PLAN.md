       - Use a debounce approach: after product webhook handling, schedule KB rebuild. Since BackgroundTasks run sequentially, the KB rebuild will run after product sync.

Trigger KB rebuild in background. Return 201 with product data.

Also count products in products table for this store. Return KBSyncStatus model + add `char_limit: 300000` and `warning_threshold: 240000` and `is_warning: bool` (char_count >= 240000) to response.

       - `POST /api/stores/{store_id}/kb/sync` — validates store owner. Checks store has elevenlabs_agent_id (return 400 if no agent). Adds `kb_service.trigger_kb_rebuild(store_id)` as background task. Returns 202 with `{"message": "KB sync started", "store_id": store_id}`.
