       - `async sync_kb_for_store(store_id: str) -> dict`: Full pipeline:
         a. Set kb_sync_status = 'syncing'
         b. Fetch all products for store from DB
         c. Transform to KB text via transform_products_to_kb_text
         d. Calculate char_count = len(kb_text)
         e. Get store's current kb_doc_id and elevenlabs_agent_id
         f. Delete old KB doc if kb_doc_id exists (handle deletion failure gracefully — log and continue)
         g. Create new KB doc via elevenlabs_service.create_kb_document_text

         i. Update stores: kb_doc_id, kb_char_count, kb_product_count, kb_last_synced=NOW(), kb_sync_status='synced'
         j. On any exception: set kb_sync_status='error', log, re-raise
         k. Return {"doc_id": ..., "char_count": ..., "product_count": ...}

       - `async generate_embeddings_for_store(store_id: str) -> int`: Generate Gemini embeddings for all products in store:
         a. Fetch products without embeddings (or all products for full re-embed)

         c. Use `self.gemini_client.aio.models.embed_content()` (async variant) with model="gemini-embedding-001", task_type="RETRIEVAL_DOCUMENT", output_dimensionality=768
         d. Batch in groups of 100 to avoid timeouts
         e. UPDATE products SET embedding = $vec WHERE id = $id AND store_id = $store_id
         f. Return count of products embedded

         a. Generate query embedding with task_type="RETRIEVAL_QUERY"
         b. Build SQL with pgvector cosine distance operator `<=>`, WHERE store_id AND embedding IS NOT NULL
         c. Add optional max_price and category filters
         d. ORDER BY embedding <=> query_vec, LIMIT
         e. Return list of dicts with id, title, description, price_min, price_max, category, product_type, similarity

       - `async trigger_kb_rebuild(store_id: str) -> dict`:
         a. Call sync_kb_for_store (handles status tracking internally)
