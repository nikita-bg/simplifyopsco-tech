1. In backend/models.py, add a new WidgetConfigResponse model at the end of the "API Request/Response Models" section:

```python
class WidgetConfigResponse(BaseModel):
    """Widget initialization config returned to embed.js"""
    has_agent: bool = False
    enabled: bool = False
    agent_id: Optional[str] = None
    widget_color: str = "#256af4"
    widget_position: str = "bottom-right"
    greeting_message: Optional[str] = None
    status: str = "unknown"
```

Note: Default color is #256af4 (project primary blue), NOT #6366f1 (the StoreSettings default is legacy indigo — the project's actual primary color per CLAUDE.md is #256af4).

2. In backend/main.py, add a new GET /api/widget/config endpoint. Place it right above the existing /api/voice/config endpoint (around line 1079). Follow the exact pattern from the research (03-RESEARCH.md Pattern 1):

- Query param: store_id (string, default "")
- No auth required (public endpoint for widget on merchant sites)
- DB query: SELECT elevenlabs_agent_id, agent_status, settings FROM stores WHERE id = $1::uuid
- Parse settings JSONB (handle str or dict)
- Return WidgetConfigResponse with:
  - has_agent: bool(agent_id)
  - enabled: settings.get("enabled", True)
  - agent_id: return the actual agent_id ONLY when agent_status == "active" AND enabled == True; otherwise null (security: don't expose agent_id for disabled agents)
  - widget_color: from settings, default #256af4
  - widget_position: from settings, default "bottom-right"
  - greeting_message: from settings
  - status: agent_status value (or "no_store_id", "store_not_found", "error" for edge cases)
- Handle edge cases: no store_id provided, store not found, DB error — all return safe defaults with has_agent=False, enabled=False
- Add response header `Access-Control-Allow-Origin: *` using FastAPI Response parameter. This is necessary because the global CORSMiddleware uses allow_credentials=True which prevents wildcard. Widget endpoints are public and don't need credentials.

3. Also add the `Access-Control-Allow-Origin: *` header to the existing /api/voice/signed-url endpoint (line 1108) using the same Response parameter pattern. This ensures the signed URL fetch from the widget on merchant sites doesn't get blocked by CORS.

Important: Do NOT change the global CORSMiddleware config. The wildcard is only for these two widget-facing endpoints. Use `response.headers["Access-Control-Allow-Origin"] = "*"` on each endpoint's Response object.
