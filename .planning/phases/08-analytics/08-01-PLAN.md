Add 4 new GET endpoints in a new "Analytics" section of main.py (add after the existing Reports section around line 1478).

**1. GET /api/analytics/overview**
Query params: `store_id` (required), `period` (optional, default "7d", accepts "7d"|"30d"|"90d")
Auth: `require_store_owner(request, store_id)`
Logic:
- Parse period to integer days (7, 30, or 90)
- Current period: conversations WHERE store_id = $1 AND started_at >= NOW() - interval '{days} days'
- Previous period: conversations WHERE store_id = $1 AND started_at >= NOW() - interval '{days*2} days' AND started_at < NOW() - interval '{days} days'
- Return: `{ total_conversations, prev_total_conversations, avg_duration_seconds, prev_avg_duration_seconds, total_duration_seconds, period }`
- avg_duration_seconds: AVG(duration_seconds) from current period, rounded to 1 decimal
- On error or no pool: return zeros

**2. GET /api/analytics/intents**
Query params: `store_id` (required), `period` (optional, default "7d")
Auth: `require_store_owner(request, store_id)`
Logic:
- Query: `SELECT intent, COUNT(*) as count FROM conversations WHERE store_id = $1 AND started_at >= NOW() - interval '{days} days' AND intent IS NOT NULL GROUP BY intent ORDER BY count DESC LIMIT 10`
- Return: `{ intents: [{ intent: string, count: number }], period }`

**3. GET /api/analytics/peak-hours**
Query params: `store_id` (required), `period` (optional, default "7d")
Auth: `require_store_owner(request, store_id)`
Logic:
- Query: `SELECT EXTRACT(HOUR FROM started_at) as hour, COUNT(*) as count FROM conversations WHERE store_id = $1 AND started_at >= NOW() - interval '{days} days' GROUP BY hour ORDER BY hour`
- Return all 24 hours (fill in zeros for hours with no conversations): `{ hours: [{ hour: 0, count: 0 }, { hour: 1, count: 5 }, ...], period }`

**4. GET /api/analytics/unanswered**
Query params: `store_id` (required), `period` (optional, default "7d")
Auth: `require_store_owner(request, store_id)`
Logic:
- The post-call webhook stores analysis in conversations. Unanswered questions are identified by sentiment="Negative" or intent containing "Unanswered" or "Unknown".
- Query: `SELECT intent, transcript, started_at, sentiment FROM conversations WHERE store_id = $1 AND started_at >= NOW() - interval '{days} days' AND (intent ILIKE '%unanswered%' OR intent ILIKE '%unknown%' OR sentiment = 'Negative') ORDER BY started_at DESC LIMIT 50`
- Return: `{ questions: [{ intent, summary (first 200 chars of transcript as text), date }], total, period }`

Helper function `_parse_period(period: str) -> int` that maps "7d" -> 7, "30d" -> 30, "90d" -> 90, default 7. Place near the analytics section.

Use parameterized queries with `$1::uuid` for store_id. Use f-string ONLY for the interval value (which is an integer from _parse_period, not user input). Example: `f"interval '{days} days'"` in the SQL string.
