<task type="auto">
  <name>Task 2: Install gitleaks pre-commit hook and configure allowlist</name>
  <files>.pre-commit-config.yaml, gitleaks.toml</files>
  <action>
1. Install the pre-commit framework (Python):
   ```bash
   pip install pre-commit
   ```

2. Create `.pre-commit-config.yaml` in the repo root with gitleaks v8.24.2:
   ```yaml
   repos:
     - repo: https://github.com/gitleaks/gitleaks
       rev: v8.24.2
       hooks:
         - id: gitleaks
   ```

3. Create `gitleaks.toml` in the repo root with an allowlist for false-positive-prone paths:
   ```toml
   [extend]
   # Use default gitleaks rules as base

   [allowlist]
   description = "Global allowlist"
   paths = [
       '''\.env\.example$''',
       '''\.planning/''',
   ]
   ```

4. Install the pre-commit hook into the local git repo:
   ```bash
   pre-commit install
   ```

5. Run gitleaks against the entire current codebase (not git history, just working tree) to verify no secrets exist in tracked files:
   ```bash
   pre-commit run gitleaks --all-files
   ```
   If any findings surface, address them (they should be false positives caught by the allowlist, or real secrets that need removal).

Do NOT use `git filter-branch` or BFG to rewrite history. The credentials are being rotated which is faster and safer.
